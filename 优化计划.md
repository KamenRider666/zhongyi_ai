# zhongyi_ai 迭代升级计划

> 参考项目：china-medical-miracle（仲景·多智能体中医问诊系统）  
> 核心策略：增量迭代，保留 zhongyi_ai 的真数据 + 真工具优势，吸收 miracle 的架构先进性  
> 编写日期：2026-07-18

---

## 背景

zhongyi_ai 与 china-medical-miracle 是同一技术栈（FastAPI + LangGraph + Qdrant + Neo4j + uv）的两个项目，共用同一套基础服务。两者的定位差异：

| | zhongyi_ai | china-medical-miracle |
|--|:--:|:--:|
| **定位** | MVP 原型 | 架构研究级产品 |
| **综合评分** | 3.2/5 | 8.6/10 |
| **最大优势** | 真实数据、工具真生效 | 架构前沿、安全极致、全栈完整 |
| **最大短板** | CORS 全开、无护栏、无中间件 | MCP 工具全 mock、零可观测 |

**本次迭代目标**：在不丢失 zhongyi_ai 真数据优势的前提下，按优先级吸收 miracle 的架构优势。

---

## 总体策略

**四个阶段，增量迭代，每个阶段产出可独立验证。**

```
第一阶段（1-2 天）: 安全加固          —— 弥合最大差距
第二阶段（2-3 天）: 架构升级          —— 中间件链 + 连接池 + 代码清理
第三阶段（1-2 周）: Agent 升级        —— 单 Agent → 多 Agent
第四阶段（1-2 周）: 基础设施增强      —— Redis/记忆/前端/多LLM/可观测
```

**核心原则**：现有的 8 个 LangChain Tool 和真实 Neo4j/Qdrant 数据是核心资产，所有升级围绕它们展开，不能变成 mock 空壳。

### Step 3.3 — Supervisor 协调器

**新增文件**：`src/agent/supervisor.py`

**参考 miracle**：Supervisor 模式 + Direct-Pass 优化。

```python
# supervisor.py 核心结构
from langgraph.graph import StateGraph, END
from langgraph.checkpoint.base import BaseCheckpointSaver

class SupervisorState(TypedDict):
    messages: list
    intent: IntentType | None
    active_agent: str
    diagnosis_result: dict | None

def create_supervisor_graph(checkpointer: BaseCheckpointSaver):
    """创建 Supervisor 多 Agent 图
    
    流程：
    用户输入 → IntentClassifier → Supervisor → 子 Agent → 结果合成 → 输出
    
    Direct-Pass: 简单问题子 Agent 直接回复，跳过 Supervisor 汇总
    """
    builder = StateGraph(SupervisorState)
    
    builder.add_node("classify_intent", classify_intent_node)
    builder.add_node("supervisor", supervisor_node)
    builder.add_node("diagnosis", diagnosis_agent_node)
    builder.add_node("prescription", prescription_agent_node)
    builder.add_node("herb", herb_agent_node)
    builder.add_node("wellness", wellness_agent_node)
    builder.add_node("synthesize", synthesize_node)
    
    builder.add_conditional_edges("supervisor", route_to_agent, {
        "diagnosis": "diagnosis",
        "prescription": "prescription",
        "herb": "herb",
        "wellness": "wellness",
        "general": "synthesize",
    })
    
    return builder.compile(checkpointer=checkpointer)
```

**4 种响应类型**（参考 miracle）：
| 类型 | 说明 |
|------|------|
| `SYNTHESIZED` | Supervisor 综合多个 Agent 结果 |
| `DIRECT_PASS` | 子 Agent 直接回复用户 |
| `PARTIAL` | 部分 Agent 结果，需补充 |
| `HANDOFF` | 从一个 Agent 移交给另一个 |

|------|:--:|------|
| 多 Agent 改造破坏现有工具 | 🔴 | 工具层不动，只改调度层；每步可回退到单 Agent |
| 连接池改造引入 bug | 🟡 | 保留 SQLite 回退路径，逐步迁移 |
| Superisor + Direct-Pass 延迟增加 | 🟡 | 简单问题走 Direct-Pass 直通，不经过 Supervisor |
| 前端改造投入产出比低 | 🟢 | 优先选项 A（Tailwind 轻量改造），选项 B 视需求决定 |
